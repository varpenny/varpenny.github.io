通过 JavaScript 触发资源下载

2018-04-03 开发 JavaScript 评论

有时候，对于一些特殊的文件类型或者特定的使用场景，通过简单的点击链接请求无法直接触发资源下载，或者在新标签页打开会被浏览器拦截，又或者在下载出错时会影响当前页面。此时，如果通过浏览器页面上右键菜单的 Save As 相关操作，则可以将指定资源下载并保存到指定位置。但是，为了节省操作时间，提高用户体验，我们往往需要一步到位。那么，JavaScript 小哥就该出场了。

使用场景

具体来说，通过 JavaScript 触发资源下载的使用场景包括以下几点：

  • 图片、PDF 等资源文件加载后浏览器会直接显示或预览而不是保存到用户指定位置
  • 资源文件的下载地址是由多个参数动态组合形成的
  • 通过打开新标签页下载资源文件，默认情况下会被浏览器拦截窗口；若在当前标签页直接请求下载资源，可能出现请求超时后影响当前页面的问题

下载图片文件

一般情况下，可以通过 <a> 标签的 download 属性下载文件。但是这个方案并不适用于非同源的资源，此时它相当于普通的超链接，点击会跳转到资源页面，而不是下载。

对于常规的图片文件，可以考虑使用 canvas 将图片转换成 base64 编码之后再通过 <a> 标签的 download 属性下载。这里需要注意的是，图片资源的 Response Headers 里需要配置 Access-Control-Allow-Origin: * 以避免调用 canvas.toDataURL 时控制台报错（blocked by CORS policy）

下载二进制文件

对于一些如 Excel 等二进制文件，可以考虑通过 <iframe> 标签的 src 属性下载。这个方案适合下载地址是由多个参数动态组合形成的情况，而且可以有效避免请求超时后影响当前页面的问题。这里需要注意的是，每次动态设置 <iframe> 标签的 src 属性时，需要添加一个类似时间戳的参数，以确保前后两次请求参数一致的情况下，后面一次也正常发起请求。
